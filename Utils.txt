Comandos actualizar backend (sirve para arrancar server y para actualizar los otros contenedores tambien)

  # 1. Stop the containers
  sudo docker-compose -f docker-compose.production.yml down

  # 2. Rebuild the backend with the fix (no cache to ensere fresh build)
  sudo docker-compose -f docker-compose.production.yml build --no-cache backend

  # 3. Start all containers
  sudo docker-compose -f docker-compose.production.yml up -d

  # 4. Monitor the backend logs to ensure it starts successfully
  sudo docker-compose -f docker-compose.production.yml logs -f backend

Comando para acceder al contenedor de mysql y actualizar base de datos

  docker exec -it taemoi-mysql-prod mysql -h 127.0.0.1 -u taemoi -p -e "Comando completo, como: ALTER TABLE taemoi_db.alumno ADD COLUMN telefono2 INT NULL;"

Comandos para eliminar todo lo existente en docker de cara a deploy limpio

  # 1. Stop the containers if are running
  sudo docker-compose -f docker-compose.production.yml down

  # 2. Eliminate the old images of containers and containers stopped
  sudo docker system prune -af

  # 3. Delete the non using volumes
  sudo docker volume prune

  # 4. Check the existing volumes
  sudo docker volume ls

  # 5. Delete all the other volumes
  sudo docker volume rm volumen1 volumen2 ... (sudo docker volume rm taemoi_db_data taemoi_nginx_cache)

Corregir fallo caída server (ERR_CONNECTION_TIMEOUT):

  Fix rápido:
  - sudo conntrack -F
  - sudo docker-compose -f docker-compose.production.yml restart frontend

  Fix completo (si el rápido no funcionó):
  - sudo docker-compose -f docker-compose.production.yml down
  - sudo conntrack -F
  - sudo docker-compose -f docker-compose.production.yml up -d

Hacer copia de seguridad de la base de datos en el servidor
